# Definition of Done

A task is considered complete only when all conditions are met:

---

## Functional

- Feature works as expected
- Edge cases handled

---

## Code Quality

- Follows architecture rules
- No business logic in controllers
- Uses services where appropriate
- Form Requests used for validation

---

## Testing

- Relevant tests pass
- No broken existing functionality

---

## Documentation

- Updated if domain or architecture changed
- Consistency maintained with implementation

---

## Formatting

- Laravel Pint executed
- Code follows project conventions

---

## Scope

- No unintended feature expansion
- Matches roadmap requirements
