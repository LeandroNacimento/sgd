# UI Guidelines

## Design Philosophy

The interface must reflect an institutional, professional system.

Clean, structured, and functional.

---

## Visual Style

- Minimalist design
- High readability
- Strong hierarchy
- Neutral colors
- No excessive decoration

---

## Layout

- Dashboard-based layout
- Sidebar navigation
- Content-first screens
- Responsive design required

---

## Components

### Tables

- Primary method for displaying documents
- Must support filtering and pagination

### Forms

- Clear grouping of inputs
- Inline validation feedback

### Buttons

- Primary / Secondary / Danger distinction
- Consistent spacing

### Status Badges

- Represent document state visually

---

## States

- Empty states must be informative
- Loading states must be clear
- Error states must be user-friendly

---

## UX Principles

- Reduce cognitive load
- Keep actions predictable
- Avoid unnecessary steps
- Prioritize efficiency for operators
